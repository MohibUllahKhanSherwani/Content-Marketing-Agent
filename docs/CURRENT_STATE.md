Resolved issue:

- `uv run pytest` originally failed when `.env` contained `WORDPRESS_DEFAULT_AUTHOR_ID=` because Pydantic tried to parse the empty string as an integer. `AppSettings` now uses `env_ignore_empty=True`.

## Locked Decisions

- Use CrewAI.
- Use CrewAI Flows for top-level orchestration.
- Use specialist Crews for strategy, content production, QA, distribution, and analytics.
- Use Azure OpenAI as the real LLM provider.
- Use `azure/gpt-4o-mini` for high-volume work by default.
- Use a stronger Azure model such as `azure/gpt-4o` for final review and strategy checks.
- Use hybrid connectors with modes: `real`, `mock`, and `auto`.
- Default external platform mode is `auto`.
- `auto` uses real API calls only when credentials, permissions, and capability checks pass; otherwise it falls back to mock.
- Human approval is required before real publishing.
- `ALLOW_REAL_PUBLISH=false` by default.
- SQLite is acceptable for demo/local development.
- PostgreSQL is the production database target.
